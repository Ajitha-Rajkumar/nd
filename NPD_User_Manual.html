Overview

This manual guides field users through the complete Network Planning & Design (NPD) workflow in the One Fiber system. The workflow covers 7 major phases from initial scope definition to construction initiation.

End-to-End Workflow:
Scope & Planning → Route Release → Survey → Design → Design Release → Design to RFC (BOQ / SOQ / CAPEX) → Initiate Construction

Scope & Planning

Route Planning in GIS

1Once sites are finalized through the CR Portal, the user marks the intercity scope in GIS.
2Open GIS and navigate to the Planned Route database.
3Plan the route using GIS Road Network and Google Maps as references.
4Update the mandatory attributes:
  • Intracity ID
  • Link ID
  • Span ID
  • Route Name
5After planning, Link ID, Span ID, and business attributes are updated based on the intercity scope.
6User team plans new links and releases them in GIS Planning DB.
What happens next: GIS Scheduler automatically checks for new SPAN/Link in Planned_route. If found, it pushes state-wise records in the workflow table. The Link status updates to "Link Record Initiated".

Scope Release

Once links have status "Link Record Initiated", they become eligible for scope release.

1Links appear on the Scope Release page on the K2 main tile.
2Select the specific link for which scope needs to be released.
3Click "Release".
4If a link crosses multiple states, a confirmation message will be displayed on screen.
5A state-wise planned route released summary mail is sent to all stakeholders.
6An all-states summary mail is also sent.
2

Route Release for Survey

After scope release, routes need to be released for survey through the One Fiber Portal.

1Login to One Fiber Portal:
https://osp.jio.com/oneinfra/login
2Click on "One Fiber Scope Release".
3Select Network Type → Intracity.
4Select your State.
5All eligible links from the planned route will be displayed.
6Click on Quick Search (marked in red) and enter your route details.
7Carefully verify:
  • Length — must match the planned route
  • Network Type — must be correct
  • All other details — cross-check with planned route
8At the bottom of the page, click "Release". Select the link and release it.
⚠ IMPORTANT: DO NOT release if details like length do not match with the planned route. Always cross-check before releasing.
After release: A mail notification is automatically triggered to all relevant users. A state-wise planned route released summary mail is also shared.

3.2 Field Survey (mDSurvey App)

TaskSystemRoleLocation
InterCity Joint SurveyGIS (Mobile)InterCity FE / MPCMMaintenance Point
InterCity Joint Survey (Passive)GIS (Mobile)State ROW ExecutiveSHQ

All scope-released links will be eligible for survey.

1Login to the mDSurvey app on your mobile device.
2Select your User Role.
3Select the Survey Span.
4Click on the Span to open it.
5Add Features: Select a feature to add from the list. You can also view or edit features you've already added.
6For each feature, fill in:
  • Construction Methodology
  • Construction Type
  • Authority
7Fill in the Structure details:
  • Feature Type & Road Side
  • Authority Type (for crossings)
  • Terrain & Type (for structures)
  • Select "OR Existing" if applicable
8Each feature will show status as "In Progress" or "Completed".
9Once all features are completed, tick the checkbox "Upload Survey".
10Click Upload to submit the survey for approval.
After upload: State-wise survey completed summary mail is sent. The survey is now pending O&M and NPE approval.

Design

After survey approval, the design phase begins using Network Engineer (NE) desktop application and automation tools.

TaskSystemRoleLocation
Create DesignNE (Desktop)NPESHQ / NHQ NPE
Design the InterCity RouteK2, GIS (Web)NPENHQ

4.1 Design Automation Tools (One-Time Setup)

Before you start: Verify survey details with planned route and designing parallel lines.
Step 1 — Add One Fiber Toolbar
1Open Network Engineer (NE) desktop application.
2Go to the Customise option.
3Add the One Fiber Toolbar.
Note: This is a one-time process. Once added, the toolbar will always be available like other toolbars.
Step 2 — Zoom to Link
4Click on "Zoom to Link" tool from the One Fiber toolbar.
5Select your Link ID.
6Survey-approved routes will be displayed. The tool marked in yellow is used for the design process.

4.2 Design Initiation

1In NE, select the Link ID / Intracity Link ID using the Zoom to Link tool.
2Click "Zoom to Link" to initiate design.
3You can now start designing the Span.

4.3 Design Link Using Custom Tool

Before starting, you must select 3 layers:

1Planned Route for the designated Span ID
2Route Walk Survey layer
3Route Walk Structure for proposed structure
Important:
• The tool works only on a single network category.
• Overlay portions must be designed manually if existing design is already available (the design tool does not consider existing design).

4.4 Setting Up Design Rules

1Set up the rules for lines required for design (e.g., Extend, Trim for overshoot and undershoot).
2The linear feature tool will auto-detect errors and correct them.
3Click "Get Details".
4Click "Create RWS Lines" to create parallel lines.
5Verify in the Table of Contents (TOC) — new layers named "Parallel Line" should be created.

4.5 Selection of Network Models

Models are selected based on survey execution methodology.

1Default parameters are pre-set for Aerial and UG (Underground) portions.
2Input the required values:
  • Cable Length
  • Span Length
3Models are pre-selected by the tool but you can change them if required.
4The dropdown will only show models applicable for the selected network type.

4.6 Design Verification

1Verify the design as per input parameters and survey parameters.
2If any issue is found, re-run the design tool and follow all design steps again from the beginning.
⚠ Always verify the design before updating in NE. Do not skip this step.

4.7 Design Completion

1Click "Create Design".
2A "Design Completed" message will appear.
3The design is now updated to the selected Work Order in NE.
Manual Steps Required After Completion:
4Duct Association: Associate duct with transmedia — this must be done manually.
5Replace Existing Structures:
  • Manhole → use MH-Dummy model
  • Pole → use Tstruct model
6Run NE Custom Tools to update boundary attributes and NE IDs for structures.
7Run NE Validation before Work Order (WO) transition.

4.8 Trace Tool for Design

1Run the Trace Tool for selection of network elements in the Work Order.
2Select the Start Structure and End Structure for the trace.
3The Trace Tool will validate the network inventory.
4Once validated, the data is sent for BOQ and SOQ generation.
Result: Once design is completed, it is available in Network Engineer and is ready to release for construction.
